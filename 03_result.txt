示例 3：一次请求中调用多个工具
============================================================
思考: 我来帮您计算100/7并查询北京和上海的天气。
LLM 要求调用 3 个工具：
  → calculator({'expression': '100/7'})
    结果: 14.285714285714286
  → get_weather({'city': '北京'})
    结果: 晴天，25°C，湿度 40%
  → get_weather({'city': '上海'})
    结果: 多云，22°C，湿度 65%

最终回复: 计算结果：
100 ÷ 7 = **14.29**（保留两位小数）

天气情况：
- **北京**：晴天，25°C，湿度 40%
- **上海**：多云，22°C，湿度 65%

本章小结：

1. 工具定义 = 函数名 + JSON Schema 参数描述
2. LLM 不执行工具，只输出"想调用什么"
3. 完整流程：用户消息 → LLM 选择工具 → 我们执行 → 结果返回 → LLM 最终回复
4. LLM 能自主判断是否需要工具，也能一次调用多个工具

关键洞察：
  LLM 就像一个"大脑"，工具就是它的"手"。
  大脑决定做什么，手去执行，结果反馈给大脑。
  这就是所有 AI Agent 的核心原理。
