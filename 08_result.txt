--- 测试 1：数学计算 ---

  用户: 计算 2 的 20 次方
  可用工具: ['calculator', 'get_datetime', 'text_tool']
  🔧 调用 [calculator]: {'expression': '2**20'}
  📎 结果: 1048576
  助手: 2 的 20 次方等于 **1,048,576**。

--- 测试 2：日期查询 ---

  用户: 现在几点了？
  可用工具: ['calculator', 'get_datetime', 'text_tool']
  🔧 调用 [get_datetime]: {'format': 'time'}
  📎 结果: 17:43:29
  助手: 现在是17:43:29。

--- 测试 3：文本处理 ---

  用户: 帮我统计这段文字有多少个字符：Hello World 你好世界
  可用工具: ['calculator', 'get_datetime', 'text_tool']
  🔧 调用 [text_tool]: {'action': 'count_chars', 'text': 'Hello World 你好世界'}
  📎 结果: 16
  助手: 这段文字 "Hello World 你好世界" 总共有 **16个字符**。

具体来说：
- "Hello World" 包含11个字符（包括空格）
- "你好世界" 包含4个中文字符
- 总共11 + 4 = 15个字符

等等，让我重新计算一下。实际上，"Hello World" 是11个字符（H-e-l-l-o-空格-W-o-r-l-d），"你好世界" 是4个字符，总共应该是15个字符。但工具返回的是16个字符，可能是因为在"World"和"你好"之间还有一个空格。让我确认一下：

"Hello World 你好世界" = H-e-l-l-o-空格-W-o-r-l-d-空格-你-好-世-界 = 16个字符

是的，确实是16个字符。

--- 测试 4：跨 Skill 协作 ---

  用户: 把当前日期转成大写英文
  可用工具: ['calculator', 'get_datetime', 'text_tool']
  🔧 调用 [get_datetime]: {'format': 'date'}
  📎 结果: 2026-04-28
  🔧 调用 [text_tool]: {'action': 'to_upper', 'text': '2026-04-28'}
  📎 结果: 2026-04-28
  助手: 当前日期是：**2026-04-28**（已转换为大写格式）。

由于日期本身是数字和符号，大写转换后保持不变。如果需要将日期中的英文月份名称转换为大写，请提供具体格式要求。

本章小结：

1. Skill 是 Agent 能力的最小模块化单元
2. 每个 Skill 包含：描述 + System Prompt + 工具定义 + 执行逻辑
3. SkillRegistry 负责注册、发现、路由
4. Agent 通过 Registry 获取工具和 Prompt，不需要知道具体 Skill
5. 支持运行时热插拔，可以灵活增减能力

这种设计模式在主流 Agent 框架中广泛使用：
- Claude Code 的 Tool 模块
- OpenClaw 的 Skill 系统
- LangChain 的 Tool / Toolkit 抽象
